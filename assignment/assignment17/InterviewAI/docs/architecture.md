# InterviewAI Architecture

InterviewAI is a small Streamlit application for practicing a short voice-based interview. It uses local tools for speech recognition and text-to-speech, with Gemini as an optional feedback engine.

## High-Level Flow

```text
User
  |
  v
Streamlit UI (app.py)
  |
  +--> Audio recording or upload
  |
  +--> src/utils.py
  |       Saves audio files and text outputs
  |
  +--> src/asr.py
  |       Local Whisper transcription
  |
  +--> src/evaluator.py
  |       Feedback orchestration
  |          |
  |          +--> src/gemini_client.py
  |          |       Gemini feedback if GEMINI_API_KEY exists
  |          |
  |          +--> Rule-based evaluator
  |                  Used when Gemini is unavailable or fails
  |
  +--> src/tts.py
          Offline pyttsx3 feedback audio generation
```

## Module Responsibilities

### app.py

Builds the Streamlit interface and controls the interview flow:

- Shows the three interview questions.
- Accepts microphone input through a Streamlit recorder component, with `st.audio_input` fallback when needed.
- Accepts uploaded WAV, MP3, or M4A files as fallback.
- Displays transcripts, final feedback, and spoken feedback.
- Stores user progress with `st.session_state`.

### src/asr.py

Handles local automatic speech recognition:

- Loads Whisper once with `functools.lru_cache`.
- Uses `WHISPER_MODEL` from `.env`, defaulting to `base`.
- Checks for FFmpeg before transcription.
- Returns a cleaned transcript string.

### src/gemini_client.py

Handles optional Gemini feedback:

- Reads `GEMINI_API_KEY` from the environment.
- Builds a structured interview-coach prompt.
- Requests concise feedback from Gemini.

### src/evaluator.py

Chooses the feedback path:

- Uses Gemini when the API key is available.
- Falls back to local rule-based scoring when Gemini is missing or fails.
- Scores based on answer length, completeness, examples, action words, filler words, and STAR-related words.

### src/tts.py

Converts final feedback text to speech:

- Uses offline `pyttsx3`.
- Saves a WAV file locally.
- Returns the generated audio path for playback in Streamlit.

### src/utils.py

Provides file helpers:

- Creates output folders.
- Saves audio inputs.
- Saves transcript and feedback text files.
- Formats transcript exports.

## Data Flow

```text
Audio input
  -> Saved in outputs/audio/
  -> Whisper transcript
  -> Stored in Streamlit session state
  -> Feedback generated by Gemini or local rules
  -> Feedback text saved in outputs/transcripts/
  -> pyttsx3 creates WAV feedback audio
  -> Audio played in Streamlit
```

## Cost Design

The app avoids paid STT and TTS services:

- Whisper runs locally.
- pyttsx3 runs offline.
- Gemini is optional and can be used with the Google AI Studio free tier.
- The app still works without Gemini by using local rule-based evaluation.

## Limitations

- Whisper requires FFmpeg.
- The first Whisper run may download the selected model.
- pyttsx3 voice quality depends on voices installed on the operating system.
- Rule-based feedback is simple and less nuanced than Gemini feedback.
